- **「複数ツール併用」が標準**。単一ツールのみは3名のみ（3076服部＝ChatGPT有料一本、6001原嶋＝Gemini無料一本、6005北＝Claude有料一本）

## 3. 第1回分析: AIツール比較実験

### 3.1 比較対象ツールの分布（n=14, 複数回答）

| ツール | 比較対象として使用 |
|---|---:|
| ChatGPT | 13/14 (93%) |
| Gemini | 12/14 (86%) |
| Claude | 10/14 (71%) |
| Google AI Studio | 5/14 (36%) |
| Copilot | 2/14 |
| Grok | 1/14 (YU ZIJIANG) |

**所感**
- **ChatGPT＋Gemini＋Claudeの3強が定番比較セット**（8名がこの3つで比較）
- **Google AI Studio**を含めた学生が意外に多い（5名）
- **Grok**を選んだのはYU ZIJIANGのみ — 英語圏ユーザーの多様性

### 3.2 選んだプロンプト（比較課題）

| カテゴリ | 人数 | 具体例 |
|---|---:|---|
| **自己紹介文生成（学会/就活/英語）** | 5 | 佐藤、原嶋、DONG、嶋根、日向 |
| **専門知識の解説** | 4 | 川野（プラズマ）、尾田（アレーアンテナ）、YU（LLMの歴史）、橋本（医療論文） |
| **推論・思考問題** | 2 | 北（フェルミ推定：コンビニ数）、満永（洗車） |
| **曖昧な一般質問** | 2 | 幕田（奨学金メリデメ）、服部（短編小説） |
| **体系的比較（複数質問）** | 1 | 森村（健康情報・未確定情報・要約の3種） |

**所感**
- **「自己紹介文生成」が最多カテゴリ**（5名）→ 講義スライドp.3の課題例をそのまま流用した可能性
- **森村（6004）の3種質問評価**が圧倒的に体系的。「曖昧な健康情報への対応力／未確定情報への対応力／要約力」を切り分けて評価
- **北（6005）のフェルミ推定**はAIの推論能力を測る良い設計
- **満永（6020情）の「洗車」**は質問意図が不明で、学生自身も「こちらの聞き方も悪い」と認めている

### 3.3 「最も使いやすいツール」の選定結果

| 選ばれたツール | 人数 | 理由の傾向 |
|---|---:|---|
| **Claude** | 5 (佐藤、幕田、満永、森村part、嶋根part) | UI・読みやすさ・表現力・納得感・選択肢提示 |
| **Gemini** | 3 (川野、橋本、森村part) | サブスク特典・信憑性・論文検索力 |
| **ChatGPT** | 2 (尾田、吉永part) | 数式解説・論理性 |
| **Google AI Studio** | 1 (日向) | 情報量が多い |
| 記載不明確 | 3 | |

**所感**
- **「Claudeが最も使いやすい」という意見が最多**（5名）。UIとテキスト品質が評価されている
- 複数ツール併用派が多く、「**使い分け**」を結論とする学生も多い（川野、佐藤、森村）

### 3.4 評価の深さ: 3段階の学生レベル

**【上級】体系的・複数観点で評価**（4名）
- **森村愉快（6004）**: 3種類の質問タイプでハルシネーションまで検出。Gemini回答に「北川進氏が化学賞を受賞」と事実確認できない記述があることを指摘 ★授業の模範例に使える
- **佐藤隼輔（3049）**: 自己紹介を3目的(学会/就活/英語)で試し、正確性・網羅性・読みやすさ・操作性・速度で多角評価
- **川野塁（5011）**: 3AIの共通点（構成順序）と差異（箇条書きvs文章）を分析、さらに「3AIの回答を統合したプラズマ総合解説」まで作成
- **嶋根蛍太郎（6016）**: 3つの用途で長文比較

**【中級】表面的だが基本要件は満たす**（7名）
- 3076服部、6001原嶋、6002YU、6005北、6012DONG、6020電尾田、6029橋本、3080日向

**【簡易】形式的・分量少**（3名）
- **幕田亮介（6019）**: 4ページのPDFだが本文はわずか、図3枚が中心
- **満永匠翔（6020情）**: 1ページ、質問内容の記述も曖昧
- **YU ZIJIANG（6002）**: pptxは4枚でほぼ空（重複提出）

| 検証した | 検証なし |

## 4. 総合傾向分析

### 4.1 学生のAI利用の全体像

```
 高度利用 ────────── 中級利用 ────────── 基礎利用
  │                    │                    │
  6016 嶋根             3076 服部            3080 日向
  6007 吉永             3049 佐藤            6029 橋本
  6012 DONG             5011 川野            6019 幕田
  6004 森村             6001 原嶋            6020 満永
                        6016 嶋根
                        6005 北
                        ...
```

**修士学生としての標準像**: ChatGPT or Gemini or Claudeの少なくとも1つに有料課金し、研究・授業・就活で毎日〜週数回使用。ハルシネーションは意識しているが検証手法は未確立。

### 4.2 分野別の傾向

| 分野 | 代表学生 | 使い方の特徴 |
|---|---|---|
| **情報科学（システム/セキュリティ）** | 嶋根、森村、吉永、DONG | Claude Code、Codex、CLIツール中心。自作パイプラインも |
| **情報科学（研究基盤）** | 原嶋、北、満永 | 論文読解・コード生成中心 |
| **電子工学** | 野口、尾田、橋本 | マイコン制御コード、回路系の理解支援、SNSコンテンツ制作など多様 |
| **物理学** | 川野 | 論文検索（SciSpaceも）、専門用語解説 |
| **応用情報** | 佐藤、服部、日向 | 標準的な修士学生利用 |

**情報科学専攻の上位層は特に高度**。Claude Code/Codex CLIのような開発者向けツールを普段使いしている学生が複数。

### 4.3 有料プラン利用率が示すこと

- **少なくとも9/12名（75%）が何らかの有料プラン加入**
- 修士学生で月額2,000-3,500円を払う覚悟 → **AIは"研究費"という認識が定着**
- 無料ツール縛りの学生（及川、原嶋）も実用に耐える程度には使えている

### 4.4 授業改善への示唆

**① ハルシネーション検証演習の必須化（優先度: 高）**
第0回では92%がハルシネーションを懸念。しかし第1回の課題で実際に検証した学生はわずか1名。「疑う姿勢」と「具体的な検証手法」の乖離が明確 → 授業2回目（本日）のハルシネーション節を実例込みで強化済み

**② プロンプト設計の「多様化」**
自己紹介生成に集中した学生が多い（5/14）。課題設計時に例示を分散させるか、「同じ質問タイプでの比較は禁止」といった制約で多様化を促すと良い

**③ ツール選定の経験則を共有**
嶋根・森村・DONG のような上級ユーザーの使い分け方（Claude=UI/説明、Codex=厳密処理、Gemini=検索、NotebookLM=論文読解）を授業で共有すると、中級層・初級層の底上げに有効

**④ 未提出25名への個別対応**
特に電子工学専攻の未提出が14名と多い。研究室単位で周知を依頼するのも手

### 4.5 次回以降の授業で使える「学生発の知見」

授業資料の実例として以下を引用すると学生の主体性が高まる：

- **森村（6004）のハルシネーション検出手法**: 「2026ノーベル賞」で検証
- **6016 嶋根の自作パイプライン**: 音声→整形→AI の3段階
- **6001 原嶋の「箇条書きすぎ問題」**: プロンプト設計で形式指定の重要性
- **3076 服部の「対話履歴の自動参照がお節介」問題**: プロンプト設計の限界
- **川野（5011）の「3AI回答を統合」手法**: クロスチェック実践例
