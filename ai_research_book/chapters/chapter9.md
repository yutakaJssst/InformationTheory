# 第9章 就活文書の作成

## 学習目標

本章を学ぶことで、以下の能力を身につけることができる。

1. **AIを活用してガクチカ・ESを効果的に作成するワークフローを習得する** — 素材の棚卸しから完成稿まで、段階的にAIを活用するプロセスを理解し、実践できるようになる。
2. **STAR法による経験の構造化技法を身につける** — Situation, Task, Action, Resultの4要素で自身の経験を整理し、説得力のある文章に変換する方法を学ぶ。
3. **研究内容を非専門家に分かりやすく伝える技法を理解する** — 専門的な研究内容を、採用担当者や面接官に伝わる言葉に変換するテクニックを習得する。

## 9.1 大学院生の就職活動とAI

### M1の就活スケジュール

大学院修士課程の学生にとって、就職活動は研究と並行して進めなければならない重要なタスクである。一般的なスケジュールは以下のようになる。

| 時期 | 活動内容 |
|:---|:---|
| M1 4月〜6月 | 自己分析、業界研究の開始 |
| M1 6月〜8月 | サマーインターンシップ応募・参加 |
| M1 9月〜11月 | 秋冬インターンシップ、OB/OG訪問 |
| M1 12月〜2月 | 本選考に向けたES準備、早期選考開始 |
| M1 3月〜 | 本選考エントリー開始（広報解禁） |
| M2 4月〜6月 | 面接・内定（選考解禁は6月1日が建前） |

この過程で、エントリーシート（ES）の作成、面接対策、研究概要の説明準備など、多くの文書作成タスクが発生する。研究に集中したいM1にとって、これらを効率的にこなすことは切実な課題である。

### 企業が大学院生に求めるもの

企業が大学院生を採用する際に注目するポイントは、大きく分けて2つある。

**1. 専門性**
- 専門知識そのものだけでなく、「深く学ぶ力」の証明
- 研究を通じて培った分析力、論理的思考力
- 技術的な課題に粘り強く取り組む姿勢

**2. 汎用スキル**
- 課題発見力：何が問題かを見抜く力
- 仮説構築力：解決策を自ら考える力
- コミュニケーション力：複雑な内容を分かりやすく伝える力
- プロジェクト管理力：計画を立てて遂行する力
- 主体性：自ら行動を起こす力

重要なのは、企業は「研究の専門知識をそのまま使いたい」のではなく、「研究を通じて培われた思考プロセスと行動特性」を評価しているということである。この理解が、効果的なガクチカ・ES作成の出発点となる。

### AIを就活で活用する意義と注意点

AIを就職活動に活用することには、明確な意義がある。

**意義**
- 自己分析の壁打ち相手として、客観的な視点を得られる
- 文章の構造化・推敲を効率的に行える
- 想定質問の生成により、面接準備の網羅性が高まる
- 専門用語の平易化など、表現の調整に威力を発揮する

**注意点**
- AIが生成した文章をそのまま提出すると、面接で自分の言葉で語れなくなる
- 複数の就活生が同じAIを使うため、没個性的な文章になりやすい
- AIは「あなたの経験」を知らない。事実の入力は必ず自分で行う必要がある
- 企業によってはAI使用を検出する仕組みを導入している場合がある

本章で提唱するアプローチは、**「AIに書かせる」のではなく「AIと一緒に磨く」** というスタンスである。自分の経験と言葉が核にあり、AIはそれを構造化し、表現を洗練する助手として機能する。

## 9.2 ガクチカとは何か

### 「学生時代に力を入れたこと」の本質

「ガクチカ」とは「学生時代に力を入れたこと」の略称で、ほぼすべての企業のESや面接で問われる定番質問である。しかし、この質問の意図を正しく理解している学生は意外に少ない。

ガクチカが問うているのは、**「何をやったか」ではなく「どのように取り組んだか」** である。華やかな経験やすごい成果が必要なわけではない。日常的な研究活動であっても、そこに課題発見→仮説→行動→成果→学びのサイクルが見えれば、十分に魅力的なガクチカになる。

### 企業が見ているポイント

採用担当者がガクチカを読む際に注目しているポイントは、以下の5つに集約される。

| ポイント | 説明 | 具体的に見ていること |
|:---|:---|:---|
| **課題発見力** | 状況の中から問題を見つける力 | 何を課題と認識したか、なぜそれが課題なのか |
| **行動力** | 自ら考えて動く力 | どのような工夫をしたか、なぜその方法を選んだか |
| **成果** | 行動の結果として何が得られたか | 定量的・定性的な成果、変化 |
| **学び** | 経験から何を学んだか | 自己成長、気づき |
| **再現性** | 入社後も同様に活躍できるか | 学んだことをどう活かすか |

特に重要なのが「再現性」である。採用担当者は「この人が入社したら、同じように課題を発見し、主体的に行動し、成果を出してくれるだろうか」という視点で読んでいる。

### 研究をガクチカにする場合の特有の課題

大学院生が研究をガクチカにする場合、学部生にはない特有の課題がある。

**課題1: 専門用語の壁**

研究内容をそのまま書くと、専門外の採用担当者には理解できない。かといって、簡略化しすぎると何をしたのか伝わらない。適切な抽象度のコントロールが求められる。

**課題2: 「やらされた」感の払拭**

研究テーマは指導教員から与えられることも多い。しかし、「先生に言われてやりました」では主体性が伝わらない。与えられたテーマの中で、自分がどのような課題意識を持ち、どう工夫したかを明確にする必要がある。

**課題3: 成果の表現**

研究の成果は論文の採択や学会発表という形になるが、それだけでは「すごさ」が伝わらない。研究の社会的意義や、その成果が何に繋がるのかまで含めて説明する必要がある。

**課題4: 個人の貢献の明確化**

研究室での研究はチームで行うことも多い。その中で自分が何を担当し、どこに独自の貢献があったのかを明確にしなければならない。

これらの課題に対して、AIは非常に有効な支援ツールとなる。次節以降で、具体的なワークフローを見ていこう。

## 9.3 AIを使ったガクチカ作成ワークフロー

ガクチカ作成を4つのステップに分け、各ステップでAIをどのように活用するかを解説する。

### Step 1: 素材の棚卸し — AIに壁打ちしてもらう

最初のステップは、自分の経験を漏れなく洗い出すことである。多くの学生は「自分には書くことがない」と思いがちだが、それは棚卸しが不十分なだけである。AIを壁打ち相手にすることで、忘れていた経験や見落としていた側面を引き出すことができる。

**プロンプト例: 経験を引き出す質問を生成**

```
# 役割
あなたは大学院生の就職活動を支援するキャリアカウンセラーです。

# 依頼
私は修士課程の学生で、ガクチカ（学生時代に力を入れたこと）の素材を探しています。
以下の情報をもとに、私の経験を深掘りする質問を10個生成してください。

# 私の情報
- 専攻: [例: 機械工学]
- 研究テーマ: [例: 自動運転における安全性評価手法の研究]
- 研究期間: [例: 2024年4月〜現在]
- その他の活動: [例: TAの経験、学会発表1回、研究室のゼミ運営]

# 質問の方向性
- 研究の中での困難や工夫に関する質問
- チームワークや他者との協働に関する質問
- 自発的に始めた取り組みに関する質問
- 失敗とそこからの学びに関する質問
- スキルアップの努力に関する質問
```

AIが生成する質問に一つずつ回答していくことで、ガクチカの素材となる「エピソードの種」が複数見つかるはずである。

**ポイント**: このステップでは、AIに文章を書かせるのではなく、AIに「質問してもらう」ことが重要である。自分で答えることで、自分の言葉で語れる素材が蓄積される。

### Step 2: STAR法での構造化

素材が見つかったら、STAR法で構造化する。STAR法は、経験を説得力のある形で整理するためのフレームワークで、就活のガクチカ作成において最も広く使われている手法である。

| 要素 | 内容 | ガクチカにおける役割 |
|:---|:---|:---|
| **S**ituation（状況） | どんな環境・背景だったか | 読み手に場面をイメージさせる |
| **T**ask（課題） | 何が課題・目標だったか | 取り組みの意義を示す |
| **A**ction（行動） | 具体的に何をしたか | 自分の主体性・工夫を示す |
| **R**esult（結果） | どんな成果が得られたか | 行動の効果を証明する |

**プロンプト例: STAR構造化を依頼**

```
# 役割
あなたは就職活動のESライティングコーチです。

# 依頼
以下の研究経験を、STAR法（Situation, Task, Action, Result）の4要素に
整理してください。各要素について、ガクチカとして効果的になるよう
ポイントも添えてください。

# 私の経験（箇条書き）
- 自動運転の安全性を評価する研究をしている
- 既存の手法では安全性の論拠が体系的に整理されていなかった
- 安全ケース（GSN）という手法に着目し、安全状態レポートと
  組み合わせる新しいアプローチを提案した
- 先行研究を50本以上読んで課題を整理した
- 国際学会（Safecomp 2025）に論文が採択された
- 研究室で初めて海外学会で口頭発表した
- 指導教員や共同研究者と週1回のミーティングで議論を重ねた

# 出力形式
S（状況）: ...
  ポイント: ...
T（課題）: ...
  ポイント: ...
A（行動）: ...
  ポイント: ...
R（結果）: ...
  ポイント: ...
```

**STAR法の実践例**

以下に、研究経験をSTAR法で構造化した例を示す。

> **S（状況）**: 自動運転システムの安全性評価に関する研究室に所属し、修士研究として安全性の論証手法に取り組んでいた。自動運転技術の実用化が進む中、安全性を体系的に保証する手法の確立が業界全体の課題となっていた。
>
> **T（課題）**: 既存の安全性評価手法では、安全性の論拠が断片的で、体系的な整理がされていなかった。「なぜこのシステムは安全と言えるのか」を論理的に説明できる枠組みを構築することが課題であった。
>
> **A（行動）**: 先行研究を50本以上精読し、既存手法の限界を整理した。その上で、安全ケース（GSN: Goal Structuring Notation）と安全状態レポートを組み合わせる新しいアプローチを考案した。指導教員・共同研究者と週1回の議論を重ね、手法の妥当性を検証した。
>
> **R（結果）**: 提案手法を論文としてまとめ、安全性工学の国際学会Safecomp 2025に投稿し、採択された。研究室として初めての国際学会口頭発表を実現した。

### Step 3: AIで文章化（400字程度）

STAR構造ができたら、これを読みやすい文章に変換する。多くのESでは400字程度の字数制限がある。

**プロンプト例: STAR構造からガクチカ文章を生成**

```
# 役割
あなたはESライティングの専門家です。

# 依頼
以下のSTAR構造を、400字以内のガクチカ文章にまとめてください。

# STAR構造
[Step 2で作成したSTAR構造をここに貼る]

# 文章の条件
- 400字以内（厳守）
- 最初の1文で「何に力を入れたか」が分かること
- 専門用語は最小限にし、非専門家でも理解できること
- 主語は「私」、ですます調
- 数字を使って具体性を出すこと
- 最後に「学び」や「今後への展望」を含めること

# 出力
ガクチカ文章（文字数カウント付き）
```

**Before（構造化前の素材メモ）**:

> 自動運転の安全性の研究をしています。GSNというのを使って安全性を評価する手法を提案しました。論文を50本くらい読みました。国際学会に通りました。

**After（AIと磨いた完成稿）**:

> 自動運転システムの安全性を論理的に証明する研究に力を入れました。修士研究として取り組んだのは、「なぜこのシステムは安全と言えるのか」を体系的に説明する枠組みの構築です。従来の安全性評価は、確認項目が断片的で全体像が見えにくいという課題がありました。私は50本以上の先行研究を精読して課題を整理した上で、安全性の論拠を木構造で可視化する手法と、システムの状態を継続的に記録する仕組みを組み合わせた新しいアプローチを考案しました。週1回の研究ミーティングでは、自ら論点を整理した資料を作成し、議論を主導しました。その結果、この成果をまとめた論文が安全性工学分野の国際学会に採択され、研究室初の海外口頭発表を実現しました。この経験を通じて、先行研究の徹底的な調査に基づく課題設定と、仮説を検証しながら新しい解決策を構築する力を身につけました。（350字）

### Step 4: AIで推敲・添削

文章化ができたら、採用担当者の視点でAIに添削してもらう。

**プロンプト例: 採用担当者視点での添削**

```
# 役割
あなたはメーカーの採用担当者（技術系）です。年間500本以上のESを読んでいます。

# 依頼
以下のガクチカ文章を、採用担当者の視点で添削してください。

# ガクチカ文章
[Step 3で作成した文章をここに貼る]

# 添削の観点
1. 【第一印象】最初の1文で興味を持てるか
2. 【課題の明確さ】何が課題だったかが明確か
3. 【行動の具体性】何をしたかが具体的に書かれているか
4. 【主体性】「自分が」やったことが伝わるか
5. 【成果】成果が具体的か（定量情報があるか）
6. 【再現性】入社後も活かせそうな能力が見えるか
7. 【読みやすさ】文章は読みやすいか、冗長な部分はないか

# 出力形式
各観点について、○/△/×の評価と、具体的な改善提案をお願いします。
改善案がある場合は、修正後の文章例も示してください。
```

**添削のサイクルのコツ**

一度の添削で完璧にしようとせず、以下のサイクルを2〜3回繰り返すと効果的である。

1. AIに添削してもらう
2. 指摘の中から納得できるものを選ぶ（全部受け入れる必要はない）
3. 自分の言葉で修正する
4. 再度AIに見てもらう

重要なのは、**AIの修正案をそのまま採用するのではなく、自分の言葉で書き直すこと**である。面接では必ずガクチカの深掘りがあり、自分の言葉で語れなければ見抜かれる。

## 9.4 研究を非専門家に伝える技法

### エレベーターピッチ（30秒で研究を説明）

エレベーターピッチとは、エレベーターに乗っている30秒程度の時間で、自分の研究を相手に伝える技法である。就職活動では、面接官に研究内容を簡潔に説明する場面で必須となる。

**30秒エレベーターピッチの構造**

1. **一言で言うと**（1文）：研究を一言で表現
2. **なぜ重要か**（1〜2文）：社会的な文脈での意義
3. **何をしたか**（1〜2文）：アプローチの概要
4. **何が分かったか**（1文）：主要な成果

**例: 自動運転安全性研究の場合**

> 「自動運転車が本当に安全かどうかを、論理的に証明する方法を研究しています。自動運転技術は急速に進歩していますが、『なぜ安全と言えるのか』を体系的に説明する方法が確立されていません。私は、安全性の根拠を木構造で整理し、システムの状態と紐づけて管理する新しい手法を提案しました。この手法により、安全性の議論が可視化され、見落としを防ぐことができます。」

### 技術的な内容の平易化テクニック

研究内容を非専門家に説明する際の、具体的なテクニックを紹介する。

**テクニック1: 日常の比喩を使う**

| 専門的な表現 | 平易な表現（比喩）|
|:---|:---|
| Goal Structuring Notation | 安全性の根拠を木構造で整理する図（家系図のようなもの）|
| 形式検証 | 数学的に「絶対に間違いがない」と証明すること |
| 機械学習モデルの過学習 | テスト勉強で丸暗記したけど応用問題が解けない状態 |
| API | ソフトウェア同士が会話するための共通言語 |

**テクニック2: 「何が嬉しいか」を先に言う**

専門家はつい「何をしたか（How）」から説明しがちだが、非専門家にとっては「それで何が嬉しいのか（Why/What）」の方がはるかに重要である。

- NG: 「GSNとSafety Status Reportを統合した手法を提案しました」
- OK: 「自動運転車の安全性を誰でも理解できる形で見える化する手法を作りました」

**テクニック3: 数字で具体化する**

抽象的な表現を避け、数字で具体化する。

- NG: 「多くの先行研究を調査しました」
- OK: 「50本以上の論文を精読し、課題を5つのカテゴリに整理しました」

### AIに「専門外の人にも分かるように」書き換えさせるプロンプト

```
# 役割
あなたは科学コミュニケーションの専門家です。

# 依頼
以下の研究概要を、理系の知識がない採用担当者でも理解できるように
書き換えてください。

# 元の研究概要
[専門的な研究概要をここに貼る]

# 書き換えの条件
- 専門用語は使わないか、使う場合は直後に平易な説明を（）で補足
- 「何が問題で、なぜ重要で、何をして、どうなったか」の順で構成
- 比喩や日常的な例えを積極的に使う
- 200字以内
- ただし、研究の本質を損なわないこと

# 出力
書き換えた研究概要と、変更した箇所の説明
```

このプロンプトのポイントは、最後の「研究の本質を損なわないこと」という条件である。平易化を求めると、AIは時に研究の核心部分まで省略してしまうことがある。この条件を明示することで、バランスの取れた書き換えが得られる。

## 9.5 ES全体でのAI活用

エントリーシートには、ガクチカ以外にも複数の設問がある。ここでは、代表的な設問ごとのAI活用法を紹介する。

### 志望動機の作成

志望動機は、ガクチカ以上に「自分の言葉」が求められる設問である。AIに丸投げすると、どの企業にも当てはまる一般的な文章になりがちなため、特に注意が必要である。

**効果的な活用法**

```
# 役割
あなたは就職活動のアドバイザーです。

# 依頼
以下の情報をもとに、志望動機の「骨格」を作成してください。
文章そのものではなく、論理構造（何を、どの順番で、なぜ書くべきか）
を提案してください。

# 情報
- 志望企業: [企業名]
- 志望職種: [職種]
- 企業の特徴（自分が魅力に感じた点）: [箇条書き]
- 自分の研究テーマ: [テーマ]
- 自分が仕事で実現したいこと: [箇条書き]
- その企業でなければならない理由: [箇条書き]

# 出力形式
志望動機の論理構造（段落ごとに、何を書くべきか、なぜそれが効果的か）
```

ポイントは、AIに「文章」ではなく「論理構造」を出力させることである。構造ができたら、自分の言葉で文章化する。

### 自己PRの作成

自己PRはガクチカと混同されがちだが、役割が異なる。

- **ガクチカ**: 具体的なエピソードを通じて能力を示す（帰納的）
- **自己PR**: 自分の強みを宣言し、エピソードで裏付ける（演繹的）

```
# 依頼
以下の強みとエピソードから、自己PR文（300字以内）の構成案を
作成してください。

# 強み
[例: 粘り強く課題に取り組む力]

# 裏付けエピソード
[例: 研究で手法がうまくいかず、3か月間試行錯誤を繰り返した経験]

# 構成案の条件
- 最初の1文で強みを宣言
- 具体的なエピソードで裏付け
- 入社後の活かし方で締める
```

### 研究概要の作成

一部の企業（特にメーカーや研究職）では、研究概要の提出を求められる。これは学会向けの概要とは異なり、非専門家を読者として想定する必要がある。

```
# 依頼
以下の研究内容を、A4用紙1枚程度の研究概要にまとめてください。
読者は、技術系の採用担当者（専門は異なる分野）です。

# 研究内容
- テーマ: [テーマ]
- 背景・課題: [箇条書き]
- アプローチ: [箇条書き]
- 結果: [箇条書き]
- 今後の展望: [箇条書き]

# 研究概要の条件
- 構成: 背景→課題→手法→結果→展望
- 図表は使わない（テキストのみ）
- 専門用語には説明を付ける
- 「この研究がなぜ社会にとって重要か」を冒頭で明確にする
- 800〜1000字
```

### 注意: AIに丸投げしない

ここまでAI活用のプロンプト例を紹介してきたが、最も重要な注意点を改めて強調する。

**ESは面接の「台本」である。**

ESに書いた内容は、面接で必ず深掘りされる。「ガクチカをもう少し詳しく教えてください」「その時、具体的にどんな工夫をしましたか？」「失敗した時、どう感じましたか？」——これらの質問に、自分の言葉で答えられなければ、ESがどんなに良くても意味がない。

AIを使って作成する場合でも、以下を心がけること。

1. **素材（事実・経験）は必ず自分で入力する** — AIは知らないことを捏造する
2. **AIの出力を鵜呑みにしない** — 自分の経験と合っているか確認する
3. **最終的な文章は自分の言葉に置き換える** — 音読して違和感がないか確認する
4. **面接で話す練習をする** — ESを見ずに、自分の言葉で3分間話せるか試す

## 9.6 面接準備でのAI活用

### 想定質問の生成

面接準備で最も効果的なAI活用は、想定質問の生成である。自分では思いつかない角度からの質問を得ることで、準備の網羅性が大幅に向上する。

**プロンプト例: ESベースの想定質問生成**

```
# 役割
あなたはメーカー（技術系）の面接官です。10年以上の面接経験があります。

# 依頼
以下のESの内容を読み、面接で聞きそうな質問を15個生成してください。
深掘り質問、意図を探る質問、価値観を問う質問をバランスよく含めてください。

# ES内容
【ガクチカ】
[ガクチカの文章を貼る]

【志望動機】
[志望動機の文章を貼る]

【自己PR】
[自己PRの文章を貼る]

# 質問のカテゴリ
- ガクチカの深掘り（5問）
- 志望動機の深掘り（3問）
- 研究内容に関する質問（3問）
- 価値観・人柄に関する質問（2問）
- 逆質問のヒント（2問）

# 各質問に対して
- 質問文
- この質問の意図（面接官が何を見ているか）
- 回答のポイント（何を含めるべきか）
```

### 模擬面接（AIに面接官役をさせる）

ChatGPTやClaudeなどの対話型AIを使って、模擬面接を行うことができる。

**プロンプト例: 模擬面接**

```
# 役割
あなたは[企業名/業界]の技術系面接官です。

# 依頼
これから模擬面接を行います。以下のルールに従ってください。

# ルール
1. 1回につき1つの質問だけをしてください
2. 私の回答に対して、簡潔なフィードバック（良かった点・改善点）を
   述べてから、次の質問に進んでください
3. 最初はアイスブレイク的な質問から始め、徐々に深い質問に進んでください
4. 全部で8問程度で終了し、最後に総合評価をしてください
5. 厳しめにフィードバックしてください（本番より厳しい方が練習になります）

# 私のES情報
[ES内容を貼る]

# 面接の種類
[一次面接/最終面接/技術面接]

それでは、面接を始めてください。
```

**模擬面接のコツ**

- 実際に声に出して回答する（タイピングではなく音声入力を使うとより実践的）
- 回答は1〜2分を目安にする（長すぎても短すぎてもNG）
- AIのフィードバックを鵜呑みにせず、「実際の面接官ならどう感じるか」を考える
- 可能であれば、友人や先輩にも模擬面接をしてもらい、AIとの違いを比較する

### 面接後の振り返り

面接が終わった後の振り返りにもAIは活用できる。

```
# 依頼
今日の面接を振り返りたいです。以下の質問に対する私の回答を分析し、
改善点を教えてください。

# 質問と回答
Q1: [面接で聞かれた質問]
A1: [自分の回答（覚えている範囲で）]

Q2: [面接で聞かれた質問]
A2: [自分の回答]

...

# 分析の観点
- 各回答の論理構造は明確だったか
- 具体性は十分だったか
- 質問の意図に正しく答えていたか
- 改善版の回答案
```

## 本章のキーポイント

1. **就活文書は「AIに書かせる」のではなく「AIと一緒に磨く」** — 素材は自分、構造化と推敲にAIを活用する
2. **STAR法は万能の構造化ツール** — 研究経験でも課外活動でも、Situation→Task→Action→Resultの順で整理すれば説得力が増す
3. **非専門家への説明は「Why/What」から始める** — 「何をしたか」より「なぜ重要か、何が嬉しいか」を先に伝える
4. **ESは面接の台本** — AIの出力をそのまま使わず、自分の言葉で語れるようにする
5. **AIを使った模擬面接は効果的だが、人間との練習も併用する** — AIは「空気」を読めないため、対人練習は別途必要

## 演習問題

### 演習9-1: 素材の棚卸し
本章のStep 1のプロンプトを使い、AIに自分の経験を引き出す質問を生成してもらいなさい。生成された質問に対してすべて回答し、ガクチカの素材となりそうなエピソードを3つ選びなさい。

### 演習9-2: STAR法による構造化
演習9-1で選んだエピソードの1つを、STAR法で構造化しなさい。AIを使ってもよいが、最終的な構造は自分の言葉で記述すること。Situation, Task, Action, Resultそれぞれについて、3〜5文で記述しなさい。

### 演習9-3: ガクチカ文章の作成と推敲
演習9-2のSTAR構造をもとに、400字以内のガクチカ文章を作成しなさい。AIに下書きを作成させた後、Step 4の添削プロンプトで少なくとも2回の推敲サイクルを行い、Before（初稿）とAfter（最終稿）を比較して提出しなさい。

### 演習9-4: エレベーターピッチの作成
自分の研究について、30秒で説明できるエレベーターピッチを作成しなさい。以下の2つのバージョンを作成すること。
- 版A: 同じ分野の研究者向け（専門用語使用可）
- 版B: 文系の採用担当者向け（専門用語不使用）
実際に声に出して読み、30秒以内に収まるか確認しなさい。

### 演習9-5: 模擬面接の実施
本章のプロンプトを使い、AIと模擬面接を実施しなさい。8問以上のやり取りを行い、AIからのフィードバックをもとに、自分の回答の改善点を3つ挙げなさい。特に「自分の言葉で語れているか」「具体性があるか」の観点で振り返りなさい。
