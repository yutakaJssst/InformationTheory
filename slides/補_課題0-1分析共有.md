# 独自の視点 ① 箇条書きすぎ問題

<div class="quote-box">
AIを利用する中で、改善を要すると感じることもある。特に困っているのは、AIの回答が<strong>箇条書きの形式で提示されることが多く</strong>、その結果として情報が必要以上に細かく断片化されてしまうことである。過度に細分化された情報は、かえって理解を困難にしてしまう場合がある。<br>
例えば、「地球温暖化の原因と影響」についてAIに説明を求めた際に、「温室効果ガスの増加」「森林破壊」「気温上昇」「海面上昇」といったように箇条書きで提示されたとする。項目ごとに分けられて整理されているが、<strong>それぞれの要素がどのように関連し合っているのか、因果関係が十分に示されない</strong>出力になることがある。
<span class="quote-author">— 6001 原嶋 康介</span>
</div>

> **プロンプトで出力形式を「文章で」と指定する価値の気づき**

---

# 独自の視点 ② 対話履歴のお節介問題

<div class="quote-box">
<strong>対話履歴の自動参照</strong> → 対話履歴を考慮して、「研究テーマに関連する情報をピックアップしますね！」と入力文に書いていない方向性で、(余計なお世話すぎる)<br><br>
<strong>対話履歴の削除</strong> → 上記で説明したような、AIの判断からユーザーの意図に合わない方向性で出力を行う。これを直す方法がない。「対話履歴を考慮しないで」と最初に言えばある程度解決するが、あくまでこれは<strong>対処療法でしかなく根本的な解決ができておらず迷惑である</strong>（「対話履歴を考慮しないで」と入力しても勝手に考慮されることもある）。
<span class="quote-author">— 3076 服部 倭</span>
</div>

> **メモリ機能の光と影** — 便利さの裏で制御しにくい問題

# 独自の視点 ③ プライバシーへの警戒

<div class="quote-box">
一番の課題は<strong>ハルシネーション</strong>（もっともらしいが間違った回答）である。研究で専門的な内容を扱う際、「分からないなら分からないと言ってほしい」と思う場面が多い。<br><br>
また、<strong>研究データをAIに入力する際のプライバシー</strong>も気になる点である。会話データが学習に使われる可能性があるため、設定でオプトアウトしているが、<strong>デフォルトで学習対象になっている仕様には不安を感じる</strong>。改善してほしい点としては、AIが自分の回答にどれくらい自信があるのかを示してくれる機能があると、ユーザー側の確認作業が楽になると考える。
<span class="quote-author">— 6004 森村 愉快</span>
</div>

> **研究データのオプトアウト設定**は全員が確認すべきポイント

# 独自の視点 ④ AI依存と自己成長のジレンマ

<div class="quote-box">
AIはうまく使えば作業を効率化できる一方で、<strong>使い方を誤ると自分の理解や成長につながりにくい</strong>という難しさもある。自分で考えて理解することを優先すればその分時間がかかるが、逆にAIに任せすぎると自分自身の力が伸びにくくなる。そのバランスを取るのは簡単ではない。<br><br>
加えて、<strong>AIが出した結果を一つずつ検証していく作業は想像以上に疲れる</strong>ため、便利さの裏で利用者側に負担が残る点も課題だと感じている。
<span class="quote-author">— 6016 嶋根 蛍太郎</span>
</div>

> 大学院生にとって本質的な問い — **「考える力」は自分で育てる**

# 独自の視点 ⑤ Agent設定の複雑さ

<div class="quote-box">

# 模範例 ③ 佐藤の多角評価（5軸）

<div class="quote-box">
研究テーマ「C2Rustの比較評価」について、以下の観点で評価：<br>
<strong>⚫ 正確性</strong>: 3ツールとも専門用語を正確に理解、文字制限遵守度も高い<br>
