## ガクチカとは

### 「学生時代に力を入れたこと」

- エントリーシート（ES）の<span class="highlight">最重要項目</span>の一つ
- 面接でも深掘りされる定番質問
- 文字数の目安: **300〜400字**が主流

### ESにおけるガクチカの位置づけ

| ES項目 | 問われていること |
|--------|----------------|
| **ガクチカ** | あなたの行動特性・思考プロセス |
| 志望動機 | 企業への理解と熱意 |
| 自己PR | 強みと再現性 |
| 研究概要 | 専門性と説明力 |

---

## 企業が見ている5つの評価軸

1. <span class="teal">**課題発見力**</span> — 何を課題と捉えたか
2. <span class="teal">**行動力**</span> — 具体的に何をしたか
3. <span class="teal">**成果**</span> — どんな結果を出したか
4. <span class="teal">**学び**</span> — 経験から何を得たか
5. <span class="teal">**再現性**</span> — 入社後にも活かせるか

### よくある誤解

- 「すごい経験」が必要 → <span class="red">**プロセスが重要**</span>
- 研究テーマが地味だからダメ → <span class="red">**取り組み方が大事**</span>
- 成果が数値化できない → <span class="red">**変化や学びで示す**</span>

# AIを使ったガクチカ作成ワークフロー

4つのStepで、ゼロから完成まで

---

<!-- _class: ai-exercise -->

## <span class="ai-badge">AI演習</span> Step 1: 素材の棚卸し

**目的**: ガクチカに使える経験を洗い出す

### 棚卸しの対象
- 研究活動（実験、学会発表、論文執筆）
- 課外活動（サークル、ボランティア、コンテスト）
- アルバイト・インターン経験
- 個人プロジェクト（開発、創作活動）

### Geminiへのプロンプト例

<div class="prompt-box">
私は○○を研究している大学院生です。<br>
就活のガクチカに使える経験を引き出すための質問を<br>
10個してください。研究活動だけでなく、<br>
課外活動やアルバイトも含めて幅広く聞いてください。
</div>

> AIに「壁打ち相手」になってもらい、自分では気づかない経験を掘り起こす

---

<!-- _class: ai-exercise -->

## <span class="ai-badge">AI演習</span> Step 2: STAR法での構造化

**STAR法** = 経験を論理的に整理するフレームワーク

| 要素 | 意味 | 質問 |
|------|------|------|
| **S** - Situation | 状況 | どんな場面だったか？ |
| **T** - Task | 課題 | 何が求められていたか？ |
| **A** - Action | 行動 | 具体的に何をしたか？ |
| **R** - Result | 結果 | どんな成果を得たか？ |

### Geminiへのプロンプト例

<div class="prompt-box">
以下の経験をSTAR法で構造化してください。<br>
各要素を2-3文で具体的に記述してください。<br><br>
[経験の概要をここに記述]
</div>

---

<!-- _class: ai-exercise -->

## <span class="ai-badge">AI演習</span> Step 3: AIで文章化

**目的**: STAR構造を自然な400字程度の文章に仕上げる

### Geminiへのプロンプト例

<div class="prompt-box">
以下のSTAR構造を元に、400字程度のガクチカを作成してください。<br><br>
条件:<br>
- 読み手は技術系企業の人事担当者<br>
- 専門用語は最小限にし、分かりやすく<br>
- 「私」は使わず「私は」で始めない<br>
- 具体的な数値やエピソードを盛り込む<br><br>
[STAR構造をここに貼り付け]
</div>

### ポイント
- 一度で完璧を目指さない → **次のStepで推敲する**
- 複数パターン生成して比較するのも有効

---

<!-- _class: ai-exercise -->

## <span class="ai-badge">AI演習</span> Step 4: AIで推敲・添削

**目的**: 複数回の推敲で文章を磨き上げる

### Geminiへのプロンプト例

<div class="prompt-box">
このガクチカを採用担当者の視点で添削し、<br>
改善点を3つ挙げてください。<br>
特に以下の観点でチェックしてください:<br>
- 具体性は十分か<br>
- 行動の主体性が伝わるか<br>
- 成果と学びが明確か<br><br>
[ガクチカ文章をここに貼り付け]
</div>

### 推敲のサイクル
1. AIに添削してもらう → 改善点を確認
2. <span class="red">**自分で**</span>修正する（ここが重要）
3. 再度AIにチェックしてもらう
4. 2-3回繰り返して完成度を上げる

---

## 良いガクチカ vs 悪いガクチカ

### <span class="red">Before（改善前）</span>

> 研究を頑張りました。大変でしたが、先生に教えてもらいながら実験をして、学会で発表しました。いい経験になりました。

<span class="small">**問題点**: 具体性がない ／ 主体的な行動が見えない ／ 成果が曖昧</span>

### <span class="green">After（改善後）</span>

> 修士研究で音声認識の精度向上に取り組んだ。先行研究の手法では雑音環境下で認識率が60%に低下する課題があった。3つの前処理手法を比較検証し、独自の組み合わせを提案した結果、認識率を85%まで改善。この成果を国内学会で発表し、優秀発表賞を受賞した。

<span class="small">**改善点**: 数値で成果を示している ／ 課題→行動→結果が明確 ／ 主体的な取り組みが伝わる</span>

---

## 注意: AIに丸投げしない

### なぜ「自分の言葉」が重要か

- <span class="red">**面接で深掘りされる**</span>: 「具体的にどうやったの？」に答えられない
- <span class="red">**嘘は見抜かれる**</span>: 面接官は何百人もの学生を見ている
- <span class="red">**AIっぽい文章**</span>: 定型的な表現が続くと不自然

### AIの正しい使い方

| <span class="green">OK</span> | <span class="red">NG</span> |
|----|-----|
| 構造化の手助け | 経験そのものを創作 |
| 表現の改善提案 | AIの出力をそのままコピペ |
| 客観的な視点の提供 | 自分で読み返さずに提出 |
| 複数案の比較検討 | 一発で完成とする |

> **AIは「壁打ち相手」であり「代筆者」ではない**

# 演習

実際にガクチカを作成してみよう

---

<!-- _class: ai-exercise -->

## <span class="ai-badge">AI演習</span> 演習1: 素材の棚卸しとSTAR構造化（20分）

### 手順

1. **Geminiを開く**（5分）
   - Step 1のプロンプトでAIに質問してもらう
   - AIの質問に答えながら、経験を書き出す

2. **経験を1つ選ぶ**（3分）
   - 最も「語れる」経験を選択
   - 研究活動がおすすめ（大学院生の強み）

3. **STAR法で構造化する**（12分）
   - Step 2のプロンプトでAIに構造化を依頼
   - AIの出力を確認し、事実と異なる部分を修正

> 時間が余った人は、2つ目の経験にも挑戦してみよう

---

<!-- _class: ai-exercise -->

## <span class="ai-badge">AI演習</span> 演習2: AIでガクチカ文章を作成・推敲（20分）

### 手順

1. **文章化**（10分）
   - Step 3のプロンプトで400字程度のガクチカを生成
   - 2-3パターン生成して比較

2. **推敲**（10分）
   - Step 4のプロンプトで添削を依頼
   - AIの指摘を参考に<span class="red">**自分で**</span>修正
   - もう一度AIにチェックしてもらう

### チェックポイント
- [ ] 400字程度に収まっているか
- [ ] STAR構造が読み取れるか
- [ ] 具体的なエピソード・数値があるか
- [ ] 自分の言葉で面接で語れるか

---

## 演習3: ペアでの相互レビュー（10分）

### 進め方

1. **隣の人とペアを組む**（1分）
2. **相手のガクチカを読む**（3分）
3. **フィードバックを伝える**（6分 = 3分×2人）

### フィードバックの観点

- <span class="green">**良い点**</span>: 具体的にどこが良かったか
- <span class="orange">**改善点**</span>: 分かりにくい部分、もっと知りたい部分
- <span class="blue">**質問**</span>: 面接官ならここを深掘りする、というポイント

> 他者の視点を取り入れることで、ガクチカの完成度が大きく上がる

---

## 研究をガクチカにするコツ

### 大学院生の最大の武器 = 研究活動

1. <span class="teal">**専門用語を言い換える**</span>
   - 「CNNによる特徴抽出」→「画像から自動で特徴を見つけるAI技術」

2. <span class="teal">**研究プロセスを強調する**</span>
   - 仮説→検証→改善のサイクルを示す

3. <span class="teal">**数値で成果を示す**</span>
   - 精度○%向上、処理時間○%削減、○件のデータ分析

4. <span class="teal">**困難と乗り越え方を入れる**</span>
   - 実験がうまくいかなかった時にどう対処したか

5. <span class="teal">**社会的意義に触れる**</span>
   - 研究が実社会でどう役立つか一言添える

---

## ES全体でのAI活用

### 同じワークフローが他の項目にも使える

| ES項目 | AIの活用方法 |
|--------|-------------|
| **志望動機** | 企業研究の整理 → 構造化 → 文章化 → 推敲 |
| **自己PR** | 強みの棚卸し → エピソード選定 → 文章化 → 推敲 |
| **研究概要** | 要点整理 → 非専門家向けに翻訳 → 文章化 → 推敲 |

### 共通するポイント
- **AIは各ステップの「加速装置」**として使う
- 最終的には<span class="highlight">自分の目と言葉</span>で仕上げる
- 企業ごとにカスタマイズすることを忘れない
