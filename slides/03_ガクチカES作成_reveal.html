ペルソナ設定は MoE の発展で意味がなくなったのでは？

学生からの問い: 「ペルソナを決めて質問するのは、すでにMoE（Mixture of Experts）の発展などで意味がないのでは？」

結論: いいえ。ただし"使い方"はアップデートが必要

3つのポイントで整理する：

  1. MoE とペルソナは「レイヤーが違う」 — 内部アーキテクチャ vs プロンプト指定
  2. ペルソナの本質は「視点・基準の設定」 — 専門知識の召喚ではない
  3. 単独では弱い、Context と Format と組み合わせて効く — ROCF全体で機能する

ペルソナが今も有効な理由

1. レイヤーが違う

  • MoE: モデル内部でトークン単位にサブネットがルーティングされる仕組み。人間に解釈できる専門単位で分かれているとは限らない
  • ペルソナ: プロンプトで立ち位置・評価基準・対象読者を指定する操作

2. 本質は「視点の切替」（自動では起きない）

ペルソナ切り替わるもの
厳しい査読者評価の厳しさ・指摘の深さ
指導教員／面接官評価軸そのもの

3. ただし単独の「○○の専門家です」は効果薄

  • 「あなたは安全性の専門家です」だけ → 弱い ／ 「ISO 26262の文脈で、5段階評価＋根拠付きで」 → 強い
  • ペルソナは「視点と評価基準の設定」と理解する。ROCF全体で機能させよう

参考になった3つの工夫

1. ROCFを少しずつ書き換えて比較する

5回連続で ROCF の要素を一つずつ変えて比較した例。「ペルソナを追加→削除」「比較指定を追加」のように何を変えると何が変わるかを切り分けて分析している

2. 禁止事項を明示する（Negative Prompting）

…改善方法を3つ提案してください。ただし、AIが提案しがちな〈高温処理〉〈薬液の使用〉〈電圧印加〉は提案に含めないでください。

→ AIが返しがちな教科書的な答えをあらかじめ封じることで、UV照射・オゾン水のような別系統の案を引き出していた

3. 自分の卒論PDFを添付して評価させる

研究テーマを文章で説明する代わりに卒論PDFを読ませると、AIが具体的な記述を踏まえて指摘や評価を返してくれる

提出物から見えた3つのハルシネーション

パターン①: 実在すると思ったら架空だった

  • 自分でちゃんと確認したら5本すべてが架空だったと報告した例
  • 反対に「すべて実在した」とまとめていたが、URL や DOI まで貼られていないため本当に検証されたかが読み手に伝わらない例も複数
教訓: 実在確認は、Google Scholar / IEEE Xplore / arXiv のヒットURL（または DOI）を貼るところまでやって初めて検証になる

パターン②: タイトルは正しいが著者だけ架空

  • 実在する論文のタイトル・掲載学会はそのままに、著者名4名がまるごと別人にすり替わっていた例
  • 「タイトルが合っている＝全部合っている」と思い込むと見落としてしまう

パターン③: 「架空論文を出すな」と書いても出てくる

  • プロンプトで「架空論文は禁止」と明示しても、架空論文が出力された
  • 指示違反は起こりうるという前提で、出力後に自分で確認することが欠かせない

鋭い指摘: モデルの自動ルーティングに気づいた学生

観察の内容

「同じgpt-5-thinking mediumを選んだのに、Zero-shot と Role 付きでは "thought for few seconds" と短く、CoT のときだけ長く考えていた。プロンプトの複雑さに応じて instant / mini モデルに振り分けられているのではないか」

ここから言えること

  • 同じモデル名を選んでも、裏で動くモデルは変わりうる（無料/有料、ルーター有無）
  • プロンプトを少し丁寧に書くだけで、上位モデルが呼ばれて回答が良くなることがある
  • ツールの選択だけでなく「どう呼び出すか」も回答の質に効いてくる

ガクチカとは

「学生時代に力を入れたこと」

  • エントリーシート（ES）の最重要項目の一つ
  • 面接でも深掘りされる定番質問
  • 文字数の目安: 300〜400字が主流

ESにおけるガクチカの位置づけ

ES項目問われていること
ガクチカあなたの行動特性・思考プロセス
志望動機企業への理解と熱意
自己PR強みと再現性
研究概要専門性と説明力

企業が見ている5つの評価軸

  1. 課題発見力 — 何を課題と捉えたか
  2. 行動力 — 具体的に何をしたか
  3. 成果 — どんな結果を出したか
  4. 学び — 経験から何を得たか
  5. 再現性 — 入社後にも活かせるか

よくある誤解

  • 「すごい経験」が必要 → プロセスが重要
  • 研究テーマが地味だからダメ → 取り組み方が大事
  • 成果が数値化できない → 変化や学びで示す

研究をガクチカにするコツ

大学院生の最大の武器 = 研究活動

  1. 専門用語を言い換える — 「CNNによる特徴抽出」→「画像から自動で特徴を見つけるAI技術」
  2. 研究プロセスを強調 — 仮説→検証→改善のサイクル（5つの評価軸の「課題発見力」「行動力」に直結）
  3. 数値で成果を示す — 精度○%向上、処理時間○%削減、○件のデータ分析
  4. 困難と乗り越え方を入れる — 実験がうまくいかなかった時にどう対処したか
  5. 社会的意義に触れる — 研究が実社会でどう役立つか一言添える
研究を語る訓練は、そのまま学振申請書・研究計画書の準備にもなる

AIを使った文書作成ワークフロー

4つのStepで、ゼロから完成まで

AI演習 Step 1: 素材の棚卸し

目的: ガクチカに使える経験を洗い出す

棚卸しの対象

  • 研究活動（実験、学会発表、論文執筆）
  • 課外活動（サークル、ボランティア、コンテスト）
  • アルバイト・インターン経験
  • 個人プロジェクト（開発、創作活動）

プロンプト例

私は○○を研究している大学院生です。
就活のガクチカに使える経験を引き出すための質問を
10個してください。研究活動だけでなく、
課外活動やアルバイトも含めて幅広く聞いてください。
AIに「壁打ち相手」になってもらい、自分では気づかない経験を掘り起こす

AI演習 Step 2: STAR法での構造化

STAR法 = 経験を論理的に整理するフレームワーク

要素意味質問
S - Situation状況どんな場面だったか？
T - Task課題何が求められていたか？
A - Action行動具体的に何をしたか？
R - Result結果どんな成果を得たか？

プロンプト例

以下の経験をSTAR法で構造化してください。
各要素を2-3文で具体的に記述してください。

[経験の概要をここに記述]

AI演習 Step 3: AIで文章化（ROCF形式）

目的: STAR構造を自然な400字程度の文章に仕上げる ／ 前回のROCFを実戦で使う

【Role】あなたは技術系企業の採用担当者で、応募者のESを書く支援をしている。
【Objective】以下のSTAR構造を元に、400字程度のガクチカを作成してください。
【Context】応募者は情報科学専攻のM1。読み手は人事担当者（非専門家）。専門用語は最小限・具体的な数値を盛り込む。
【Format】400字±50字／改行なしの段落形式／「私は」で始めない／2パターン生成して比較

[STAR構造をここに貼り付け]
ROCFで構造化することで、複数回試行しても結果が安定する

AI演習 Step 4: AIで推敲・添削（ペルソナ活用）

目的: 複数回の推敲で磨く ／ 前回の「ペルソナ設定」を実戦で使う

【Role】あなたは大手IT企業で年間500人以上のESを読んでいる厳しい採用担当者です。
【Objective】以下のガクチカを添削し、改善点を3つ挙げてください。
【Context】特に「具体性」「行動の主体性」「成果と学び」の3観点でチェックしてください。
【Format】各改善点を「指摘／なぜ問題か／具体的な書き換え案」の3点セットで提示

[ガクチカ文章をここに貼り付け]

推敲サイクル: ①AI添削 → ②自分で修正（ここが重要）→ ③再度AIチェック → ④2-3回繰り返す ／ ペルソナを「指導教員」「面接官」に変えて多角的に評価するのも有効

良いガクチカ vs 悪いガクチカ

Before（改善前）

研究を頑張りました。大変でしたが、先生に教えてもらいながら実験をして、学会で発表しました。いい経験になりました。

問題点: 具体性がない ／ 主体的な行動が見えない ／ 成果が曖昧

After（改善後）

修士研究で音声認識の精度向上に取り組んだ。先行研究の手法では雑音環境下で認識率が60%に低下する課題があった。3つの前処理手法を比較検証し、独自の組み合わせを提案した結果、認識率を85%まで改善。この成果を国内学会で発表し、優秀発表賞を受賞した。

改善点: 数値で成果を示している ／ 課題→行動→結果が明確 ／ 主体的な取り組みが伝わる

注意: AIに丸投げしない

なぜ「自分の言葉」が重要か

  • 面接で深掘りされる: 「具体的にどうやったの？」に答えられない
  • 嘘は見抜かれる: 面接官は何百人もの学生を見ている
  • AIっぽい文章: 定型的な表現が続くと不自然

AIの正しい使い方

OKNG
構造化の手助け経験そのものを創作
表現の改善提案AIの出力をそのままコピペ
客観的な視点の提供自分で読み返さずに提出
複数案の比較検討一発で完成とする
AIは「壁打ち相手」であり「代筆者」ではない

演習

実際にガクチカを作成してみよう

AI演習 演習1: 素材の棚卸しとSTAR構造化（20分）

手順

  1. AIツールを開く（5分）
    • Step 1のプロンプトでAIに質問してもらう
    • AIの質問に答えながら、経験を書き出す
  2. 経験を1つ選ぶ（3分）
    • 最も「語れる」経験を選択
    • 研究活動がおすすめ（大学院生の強み）
  3. STAR法で構造化する（12分）
    • Step 2のプロンプトでAIに構造化を依頼
    • AIの出力を確認し、事実と異なる部分を修正
時間が余った人は、2つ目の経験にも挑戦してみよう

AI演習 演習2: AIでガクチカ文章を作成・推敲（20分）

手順

  1. 文章化（10分）
    • Step 3のプロンプトで400字程度のガクチカを生成
    • 2-3パターン生成して比較
  2. 推敲（10分）
    • Step 4のプロンプトで添削を依頼
    • AIの指摘を参考に自分で修正
    • もう一度AIにチェックしてもらう

チェックポイント

[ ] 400字程度に収まっているか ／ [ ] STAR構造が読み取れるか ／ [ ] 具体的なエピソード・数値があるか ／ [ ] 自分の言葉で面接で語れるか

演習3: ペアでの相互レビュー（10分）

進め方

  1. 隣の人とペアを組む（1分）
  2. 相手のガクチカを読む（3分）
  3. フィードバックを伝える（6分 = 3分×2人）

フィードバックの観点

  • 良い点: 具体的にどこが良かったか
  • 改善点: 分かりにくい部分、もっと知りたい部分
  • 質問: 面接官ならここを深掘りする、というポイント
他者の視点を取り入れることで、ガクチカの完成度が大きく上がる

自己アピール文書全般でのAI活用

同じ4ステップが、就活以外の文書にも使える

文書AIの活用方法
志望動機企業研究の整理 → 構造化 → 文章化 → 推敲
自己PR強みの棚卸し → エピソード選定 → 文章化 → 推敲
研究概要 (ES)要点整理 → 非専門家向けに翻訳 → 文章化 → 推敲
学振DC申請書業績棚卸し → 研究背景・着想・計画の構造化 → 文章化 → 推敲
研究計画書先行研究整理 → 課題・目的・方法の構造化 → 文章化 → 推敲
奨学金・推薦状素材エピソード棚卸し → 強みとの紐付け → 文章化 → 推敲

共通するポイント

  • AIは各ステップの「加速装置」。最終的には自分の目と言葉で仕上げる
  • 提出先ごと（企業・学振・指導教員）にカスタマイズする
